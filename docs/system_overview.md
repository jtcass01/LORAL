# System Overview

## Problem Statement
Infrared communication systems are commonly used for short-range wireless communication, but their effectiveness as a sensing modality for robot localization has not been well characterized. Lunar permanently shadowed regions, such as those found near the lunar poles, receive little to no direct sunlight, potentially reducing interference from ambient light sources and making them favorable environments for infrared-based sensing and communication systems. This project investigates whether a network of fixed IR beacons can simultaneously support wireless communication and robot localization in a controlled low-light environment. The primary sensing challenge is determining whether received IR signal characteristics can be used to accurately estimate a robot’s position and orientation while maintaining reliable communication performance.


## Application and Sensors:
The system consists of three fixed IR beacon towers and one mobile robot operating in a low-light environment. Each beacon transmits a uniquely modulated IR signal using a TSAL6200 emitter at different carrier frequencies (e.g., 10 kHz, 18 kHz, 27 kHz), enabling beacon identification via frequency-division separation. The robot uses a BPW34 photodiode with MCP6002 amplification to detect signals. The Raspberry Pi Pico 2 W samples the signal (~100 kS/s), and frequency components are extracted using a Goertzel algorithm to estimate beacon-specific signal strength for communication and localization. A servo motor enables robot rotation to evaluate directional signal variation, and a BNO080 IMU provides ground-truth orientation. Q-learning will be applied to optimize robot orientation. Localization is performed using beacon signal strength comparison and geometric estimation. Performance is evaluated using data rate, BER, packet success rate, signal strength, and localization error in a 3 m × 3 m area. Key constraints include distance attenuation, angular sensitivity, and ambient light noise.
- IR Receiver (built with BPW34 Photodoide and MCP6002 Operation Amplifier and resistors)
- IR Transmitter (built with TSAL6200 IR LED and 2N2222 Transistor with resistors)
- Servo Motor (SG90 Micro)
- IMU (BNO085)

## Power Considerations
- The Pico, LED, and motor will be powered by a 5V 2A supply.
    - No need to upregulate the signal to the motor. SG90 can handle 3.3V signal.
- Each sensor is powered from the RPI Pico 2W 3.3V pin.
    - Currents are well within 3.3V rail limit of 150 mA

## Communication and Wiring Considerations
- Each IR receiver circuit is sampled using an analog-to-digital converter (ADC) input on the Raspberry Pi Pico 2W
- IR transmitter is controlled through a GPIO pin and driven by a transistor switching stage.
- The SG90 motor is controlled suing a PWM signal generated by a GPIO pin.
- The BNO085 includes a UART, SPI, and I2C interfaces. (UART is used here)
- Each agent is assigned a unique IR carrier frequency (e.g., 10 kHz, 18 kHz, 27 kHz and 35 kHz). 
- RPI Pico 2W will sample ADCs at 100 kHz
- Digital data is transmitted using On-Off Keying (OOK), where the presence of the carrier during a symbol interval represents a logical ‘1’ and the absence of the carrier represents a logical ‘0’
- Received IR signals are digitized and processed using the Goertzel algorithm to detect pre-chosen frequencies.
